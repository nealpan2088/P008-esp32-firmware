# ESP8266 传感器 — 完整部署流程

## 一、硬件清单

| 物料 | 数量 | 备注 |
|:----|:---:|:-----|
| NodeMCU V3 (CH340 / Type-C) | 1 | ESP8266 主控 |
| DS18B20 防水型不锈钢探头 | 1 | 冷柜温度 |
| SCT-013-000 开合式电流互感器 | 1 | 10A/1V，夹在电源线上 |
| MC-38 门磁开关（常闭型） | 1 | 柜门状态 |
| 100kΩ 电阻 1/4W | 2 | SCT-013 分压偏置 |
| 4.7kΩ 电阻 1/4W | 1 | DS18B20 上拉 |
| 86 明装盒 | 1 | 外壳 |
| Micro USB 线 / Type-C 线 | 1 | 供电 |
| 5V/1A 充电头 | 1 | 供电 |

## 二、环境搭建（电脑上）

### 方法 A：VS Code + PlatformIO 插件（推荐）

1. 安装 VS Code
2. 扩展 → 搜索 **PlatformIO IDE** → 安装
3. 重启 VS Code
4. `文件 → 打开文件夹` → 选择 `hardware/esp8266-sensor`
5. 等待 PlatformIO 初始化（第一次会下载工具链，约 2-5 分钟）

### 方法 B：命令行 PlatformIO

```bash
pip install platformio
cd hardware/esp8266-sensor
```

## 三、配置

### 3.1 在 P008 后台创建设备

```
登录 https://zghj.openyun.xin
→ 设备管理 → 添加设备
  - 名称: 后厨冷柜传感器
  - 序列号: SENSOR-001     ← 自己编一个
  - 设备密钥: sk-001       ← 随便输
  - 类型: 温湿度
  - 所属门店: 演示冷库
→ 保存 → 记录 serial 和 deviceKey
```

### 3.2 配置固件

```bash
cd hardware/esp8266-sensor
cp include/config.h include/config.local.h
```

编辑 `include/config.local.h`：

```c
#define WIFI_SSID       "你家WiFi名字"
#define WIFI_PASSWORD   "你家WiFi密码"
#define API_BASE_URL    "https://zghj.openyun.xin/api/v1"
#define DEVICE_SERIAL   "SENSOR-001"       // 后台创建的 serial
#define DEVICE_KEY      "sk-001"           // 后台设置的 deviceKey
```

### 3.3 可选：选择 SENSOR_TYPE

```c
#define SENSOR_TYPE     "DS18B20"  // 冷柜模式（默认）
// #define SENSOR_TYPE  "DHT22"    // 环境模式（仓库/后厨）

// 如果选 DHT22，接线改为 D4(GPIO2)
```

## 四、接线

```
NodeMCU V3              传感器
┌─────────────────────────────────┐
│                                 │
│ D3 (GPIO0) ─── 黄 ─── DS18B20   │  + 4.7kΩ → 3.3V (上拉)
│                                 │
│ D1 (GPIO5) ─── COM ─── MC-38    │
│ GND         ─── NC  ─── MC-38   │
│                                 │
│ A0          ─── 分压中点 ─── SCT-013 信号线
│ 3.3V ─── 100kΩ ─┬─ A0          │
│                 │              │
│ GND   ─── 100kΩ ─┘  ─── SCT-013 GND
│                                 │
│ 3.3V ─── DS18B20 红            │
│ GND   ─── DS18B20 黑            │
│ GND   ─── SCT-013 GND          │
│                                 │
│ VU (5V) ─── 不接                │
│ Micro USB ─── 充电头            │
└─────────────────────────────────┘
```

## 五、刷机

### 5.1 首次刷机（USB）

1. **用 Type-C 线连接 NodeMCU 到电脑**
2. 在 VS Code 中点击底栏 `→ (箭头)` 或运行：

```bash
# 普通模式（深度睡眠）
pio run -t upload

# OTA 调试模式（首次必须 USB）
pio run -t upload --environment nodemcuv2-ota
```

3. 等待编译上传完成
4. 打开串口监视器验证：

```bash
pio device monitor
```

应该看到：
```
========================================

设备: SENSOR-001  |  温度传感器: DS18B20
========================================
[WiFi] 连接 你家WiFi ...
[WiFi] 已连接, IP: 192.168.x.x
[DS18B20] 温度=2.5°C
[SCT-013] 电流=0.45A
[门磁] 门关
[HTTP] POST https://zghj.openyun.xin/api/v1/devices/SENSOR-001/data
[HTTP] 状态码: 200
[主流程] 完成，进入深度睡眠
```

### 5.2 后续升级（OTA 无线）

```bash
# 先刷一次 OTA 模式的固件（必须 USB 刷）
pio run -t upload --environment nodemcuv2-ota

# 以后改代码后，无线 OTA 刷机
pio run -t upload --upload-port 192.168.x.x
```

OTA 模式启动后会等待 10 分钟，在此期间电脑可以连接上传新固件。

## 六、安装到冷柜

```
1. 把 ESP8266 塞进 86 明装盒
2. DS18B20 探头从门缝伸入冷柜内部
3. 关门压住 DS18B20 的线
4. SCT-013 夹在冷柜电源线上（随便一根）
5. MC-38 门磁粘在柜门和门框上
6. 插上充电头
7. 盖上 86 盒盖子
8. 等待 5 分钟 → 后台看到数据
```

## 七、绑定到设备管家

```
登录 https://zghj.openyun.xin
→ 店铺管理 → 点击"演示冷库"
→ 设备管家 → 添加设备
  - 名称: 后厨冷柜
  - 分类: 冷柜
→ 绑定传感器 → 选择刚刚创建的物理设备
→ 设置报警规则
  - 温度 > 8℃ 预警
  - 温度 > 12℃ 报警
  - 电流 < 0.08A 检测关机
  - 门磁 检测门未关好
→ 保存
```

## 八、验证

等待 5 分钟后：

1. ✅ 查看设备管家页面 — 是否显示温度
2. ✅ 开门测试 — 5 分钟后触发门未关好报警
3. ✅ 拔掉冷柜电源 — 5 分钟后触发压缩机故障报警
4. ✅ 微信/钉钉是否收到通知

## 九、常见问题

| 问题 | 原因 | 解决 |
|:----|:----|:-----|
| 串口输出乱码 | 波特率不对 | `pio device monitor -b 115200` |
| WiFi 连不上 | 密码错/信号弱 | 检查 SSID 和密码，ESP8266 只支持 2.4G |
| DS18B20 读不到 | 上拉电阻漏了 | 黄线到 3.3V 加 4.7kΩ 电阻 |
| 电流一直是 0 | 分压电阻没焊 | 检查两个 100kΩ 到 A0 |
| HTTP 401 | deviceKey 不对 | 检查 config.local.h 里的 key |
| HTTP 404 | serial 不对 | 检查后台设备和 config 中的 serial |
| 不上报数据 | 没深度睡眠到 5 分钟 | 正常，5 分钟上报一次 |
