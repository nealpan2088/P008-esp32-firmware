# ESP8266 传感器固件 — 部署指南

## 目录结构
```
hardware/
└── esp8266-sensor/
    ├── platformio.ini          # PlatformIO 项目配置
    ├── include/
    │   ├── config.h            # 默认配置模板
    │   └── config.local.h      # 实际配置（git ignored，自己创建）
    └── src/
        └── main.cpp            # 主程序
```

## 传感器选择

通过 `config.local.h` 中的 `SENSOR_TYPE` 选择：

| SENSOR_TYPE | 温度传感器 | 湿度 | 适用场景 |
|:-----------:|:----------:|:----:|:---------|
| `"DS18B20"` | DS18B20 防水探头 | ❌ 无 | **冷柜/冰柜/冷冻库** |
| `"DHT22"`   | DHT22 模块 | ✅ 有 | 仓库/后厨/机房 |

**冷柜场景推荐 DS18B20：**
- 防水不锈钢探头，可从门缝伸入柜内
- 关门直接压住线，无需打孔
- 精度 ±0.5℃，冷柜够用
- 支持一根总线挂多个探头

## 首次部署步骤

### 1. 安装 PlatformIO
```bash
pip install platformio
```

### 2. 创建本地配置
```bash
cp include/config.h include/config.local.h
```

### 3. 修改配置
编辑 `include/config.local.h`：

```c
#define WIFI_SSID       "你的WiFi名称"
#define WIFI_PASSWORD   "你的WiFi密码"
#define DEVICE_SERIAL   "设备序列号"       // 先在P008后台创建
#define DEVICE_KEY      "设备密钥"         // 后台创建时自动生成/手动设置
```

### 4. 编译并上传
```bash
cd hardware/esp8266-sensor
pio run -t upload
```

### 5. 查看串口日志
```bash
pio device monitor
```

## 接口协议

设备每 5 分钟上报一次数据到后端：

```
POST /api/v1/devices/{serial}/data
Headers:
  X-Device-Key: {deviceKey}
  Content-Type: application/json

{
  "temp": 2.5,          // 温度 (°C)
  "humidity": 68,       // 湿度 (%)
  "battery": 3.3,       // 电压 (V)
  "otherData": {
    "currentA": 0.45,   // 电流 (A), SCT-013
    "doorOpen": false   // 门磁状态 (true=开门)
  }
}
```

## 硬件接线

| ESP8266 引脚 | 连接设备 |
|:------------:|:---------|
| D3 (GPIO0) | DS18B20 DATA（带 4.7kΩ 上拉到 3.3V） |
| D4 (GPIO2) | DHT22 DATA（仅 DHT22 模式，带 4.7kΩ 上拉） |
| D1 (GPIO5) | 门磁传感器（INPUT_PULLUP, LOW=开） |
| A0 (ADC)   | SCT-013 输出（经 100kΩ×2 分压偏置） |
| 3.3V       | DS18B20 VCC / DHT22 VCC / 门磁 VCC |
| GND        | 所有传感器 GND |

### DS18B20 接线（冷柜模式）

```
DS18B20 (不锈钢探头)     ESP8266
红 (VCC)         →     3.3V
黄 (DATA)        →     D3 (GPIO0)   + 4.7kΩ → 3.3V
黑 (GND)         →     GND
```

**安装：探头从冷柜门缝伸入柜内，关门压住线，无需打孔。**

### 门磁传感器
```
门磁         ESP8266
COM    →    D1 (GPIO5)
NC     →    GND
(门关闭时 COM-NC 导通 → GPIO5=GND=LOW=门开？)
```
注意：门磁通常是**常闭型**（门关=导通），GPIO5 上拉，所以门关时读到 LOW，门开时读到 HIGH。
但代码中 `readDoor()` 用 `digitalRead(DOOR_PIN)` 上拉，实际逻辑取决于门磁接 NC 还是 NO。

### SCT-013 电流互感器
SCT-013 输出是交流信号（0-1V），需要偏置到 ESP8266 ADC 的中间点（1.65V）：

```
SCT-013 输出 (3.5mm 插头):
  尖端  →  分压电路 → A0 (ADC)
  环    →  GND

简易偏置电路:
  3.3V ─┬─ 100kΩ ─┬─ A0
        │          │
        └─ 100kΩ ─┘
                   │
                  GND
```
（两颗100kΩ电阻分压，ADC 静置时读到 ~1.65V，SCT-013 信号叠加在这个偏置上）

### USB 充电
ESP8266 NodeMCU V3 自带 Micro USB 口，直接插手机充电头即可：
- 5V/1A 充电头（随便一个旧手机充电头）
- Micro USB 线
- ESP8266 板载稳压到 3.3V，功耗 ~80mA

## 首次使用流程

1. 登录 P008 后台 → 设备管理 → 添加物理设备
2. 记录分配的 serial 和 deviceKey
3. 填写到 `config.local.h`
4. 编译上传到 ESP8266
5. 插电 → 自动连接 WiFi → 上报数据 → 深度睡眠
6. 每 5 分钟重复一次
7. 后台看到设备上线 → 绑定到"设备管家"（冷柜/空调等）
